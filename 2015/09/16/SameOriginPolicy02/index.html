同源策略的限制 - Cookie无法读取

Cookie 是服务器写入浏览器的一小段信息, 只有同源的网页才能共享; 但是, 如果两个网页一级域名相同, 只是二级域名不同的话, 浏览器就允许通过设置document.domain来共享 Cookie ;

举例来说，A网页是 http://w1.example.com/a.html，B网页是 http://w2.example.com/b.html, 那么只要设置相同的document.domain，两个网页就可以共享Cookie,
现在, A网页通过脚本设置一个 Cookie, B网页就可以读到这个 Cookie :
A: www.test.com/index.html

B: test.test.com/index.html

注意, 这种方法只适用于 Cookie 和 iframe 窗口, LocalStorage 和 IndexDB 无法通过这种方法来规避同源政策，而要使用下文介绍的PostMessage API ;

另外，服务器也可以在设置Cookie的时候，指定Cookie的所属域名为一级域名, 比如 .example.com ;

这样的话，二级域名和三级域名不用做任何设置，都可以读取这个Cookie ;
