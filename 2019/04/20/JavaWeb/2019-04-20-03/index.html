第一个Servlet类

  1. 创建JavaWeb项目: Javaee -> Web Application

  2. 第一个Servlet类（实现 Servlet 接口）

  3. 在项目的 web/WEB-INF/web.xml 中配置 Servlet, 需要告诉 tomcat, 当浏览器访问 哪个资源名可以映射到哪个 Servlet类

  4. 先关注 Servlet 接口 中的 service 这个提供服务的方法 (在IDE中集成了tomcat, 可以直接在中对项目虚拟目录进行配置)

  5. 然后访问 localhost:8080/demo01/demo01 即可看到idea控制台打印了 “Hello Servlet!” (注意: 如果你在idea中配置tomcat时, Edit configur.. -> demploy -> Application Context 中配置了虚拟路径)
    访问时需要加上虚拟路径, 即 localhost:8080/demo/demo01

servlet 执行原理

  1. localhost:8080/demo01/demo01 先找到服务器, 当服务器接收到客户端浏览器的请求后, 会解析url路径, 先找到虚拟目录对应的项目, 最后获取访问的Servlet的资源路径名 demo01

  2. 根据资源路径名, 去web.xml中对应的 <url-pattern>, 找到映射的 <servlet-class> 全类名

  3. tomcat会将Servlet类的字节码文件加载进内存, 并且创建其对象

  4. 最后调用其方法

servlet 中的方法

  1. init 初始化方法: 在Servlet被创建时执行, 即第一次访问资源时执行 (只会执行一次)

    • servlet 的 init() 方法只执行一次, 说明 Servlet在内存中只存在一个对象, Servlet是单例, 多用户同时访问时, 可能存在线程安全问题
    • 因此, 我们需要注意: 尽量不要再servlet中定义成员变量, 可以定义局部变量, 即使定义了成员变量, 也不要对其进行修改, 只获取就行了
      (另外, 需要注意如果强制给service代码加同步代码块, 效率影响会非常大)

  2. service 提供服务的方法: 每次servlet被访问都会执行

  3. destroy: 在servlet被杀死前执行, 即服务器正常关闭前执行一次, 一般用于释放资源

  4. getServletInfo: 获取servlet的一些信息 版本, 作者 等等; 一般不会实现

@WebServlet 注解 相关配置

  1. @WebServlet 注解的 urlPatterns 参数可以给一个 Servlet 配置多个访问路径, 如 @WebServlet({"/demo03", "/demo003"})

  2. urlPatterns 的定义规则

    • /xxx
    • /xxx/xxxx : 多层路径 ( @WebServlet({"/d1/d2/d3", "/demo4/*"} )
    • /* : 表示写什么都可以访问到该Servlet (不常见)
    • *.xxx : 不能加 /
