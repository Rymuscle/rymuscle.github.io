2019-05-11

Spring 概述

  1. Spring 现在已不再简单是一个框架了, 在 Spring 的项目中主要有 SpringFramework(也就是我们常说的Spring)、SpringBoot、SpringCloud、SpringData、SpringIO 等等 (详情请见 官网)
    (SpringFramework 已于2017年9月发布了5.0(GA)版, 接下来的内容都是基于Spring5进行的)

  2. SpringFramework (以下简称 Spring)

    • 是一个开源的 控制反转(Inversion of Control, IoC) 和 面向切面(AOP) 的容器框架, 它的主要目的是简化企业开发
    • Spring由大量的模块组成, 其中主要包括: Core、Beans、Context、AOP、Web、ORM、JDBC 等等, 在这些组件中, 主要以 Core、Beans、Context 为核心, Spring框架通过该组件实现依赖注入与控制反转, 使得设计和测试松散耦合, 极大提高了编程效率
    • 在Spring 仓库可以下载 spring-framework-5.1.4.RELEASE-dist.zip
      dist：该文件夹下包含Spring jar包、文档、项目等内容
      docs: 该文件夹下包含Spring的相关文档、开发指南及API参考文档
      schema: 里面包含了Spring所用到的xsd文件

  3. 使用 Spring 为我们带来了很大的便捷, 它为企业级开发提供了丰富的功能 :

    • 控制反转(IOC)：Spring使用控制反转技术实现了松耦合。依赖被注入到对象, 而不是创建或寻找依赖对象降低组件之间的耦合度, 实现软件各层之间的解耦
    • 面向切面编程(AOP): Spring支持面向切面编程, 把应用的业务逻辑与系统的服务分离开来, 利用AOP很容易实现如 权限拦截、运行期监控 等功能
    • 众多服务: 如 事务管理服务、消息服务等等 (当我们使用容器管理事务时, 开发人员就不再需要手工控制事务, 也不需要处理复杂的事务传播, 声明式事务的支持, 可以将我们从单调烦闷的事务管理代码中解脱出来, 通过声明式方式灵活的进行事务的管理, 提高开发效率和质量)
    • 众多辅助及工具类, 使用这些类能够加快应用的开发 (如 JdbcTemplate、HibernateTemplate、StringUtils.. )
    • 方便集成各种优秀框架: Spring 可以降低各种框架的使用难度，提供了对各种优秀框架(Struts、Hibernate、Hessian、Quartz 等)的直接支持
    • MVC框架：现在 Spirng 已不再简单是一个框架了, Spring MVC 是一个非常好的MVC框架, 可以替换其他web框架诸如Struts
    • 方便程序的测试：Spring写出来的代码非常容易做单元测试, 可以采用依赖注射(Dependency Injection)将测试的数据注射到程序中
    • 模块分离：Spring框架是由模块构成的, 虽然已经太多的包和类, 但它们都按照模块分好类了, 你可以只考虑你会用到的模块, 而不用理其他的模块 (轻量级)
    • Java 源码是经典学习范例: Spring 的源代码设计精妙、结构清晰、匠心独用，处处体现着大师对 Java 设计模式灵活运用以 及对 Java 技术的高深造诣。它的源代码无意是 Java 技术的最佳实践的范例

  4. 小结: Spring可以做很多事情, 它为企业级开发提供了丰富的功能, 但是这些功能的底层都依赖于它的两个核心特性, 即 依赖注入(dependency injection, DI) 和 面向切面编程(aspect-oriented programming, AOP)

  5. 轻量级?
    Spring框架提供了很多服务, 但这些服务并不是默认为应用打开的, 应用可以按需指明使用的服务
    如果应用使用的服务很少, 如只使用了spring核心服务, 那么我们可以认为此时应用属于轻量级的, 如果应用使用了spring提供的大部分服务, 这时应用就属于重量级

Spring 体系架构

  1. 下载并解压 spring-framework-5.1.4.RELEASE-dist.zip 后, 可以看到 Spring Framework 有 docs、libs 和 schema 文件夹

  2. 其中libs文件夹下是所需要的jar包(从Spring 5 RELEASE版本开始, Spring 提供了21个模块, 可以看到每一模块都有三个jar包, 分别为类库、文档和源码, 所以总共会有63个jar包), 在项目中引用Spring Framework时, 应该按需引用所需的类库

  3. 下图是从 spring-framework-5.1.4.RELEASE 解压包下的 docs/spring-framework-reference/images 中拿到的

  4. Spring 总共大约有 21 个模块, 而这些组件被分别整合在 核心容器(Core Container)、AOP(Aspect Oriented Programming)、设备支持(Instrmentation)、 数据访问及集成(Data Access/Integeration)、Web、报文发送(Messaging)、Test 7个模块集合中

Spirng 各模块之间的依赖关系

该图是 Spring5 的包结构, 可以从中清楚看出 Spring 各个模块之间的依赖关系

另外, 因为 spring-core 依赖了 commons-logging, 而其他模块都依赖了 spring-core, 所以整个spring框架都依赖了commons-logging
日志框架有多种, 也不一定使用commons-logging, 如果有自己的日志实现如 log4j, 可以排除对 commons-logging 的依赖, 否则会编译报错
