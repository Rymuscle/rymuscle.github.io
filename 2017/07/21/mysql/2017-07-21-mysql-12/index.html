12. MySQL5.7.23 编译安装?

2017-07-21
  • Linux

  1. 本地下载mysql5.7.23, vagrant挂载目录到虚拟机内部 /test, 将Tarball包移动到 /usr/local/src 目录下, 并 tar -zxvf mysql-5.7.23.tar.gz 解压;

  2. mysql源码编译所需的依赖安装, 可以参考

    • https://dev.mysql.com/doc/refman/5.7/en/source-installation.html
    • https://dev.mysql.com/doc/refman/5.7/en/binary-installation.html#binary-installation-layout:

  3. 依赖安装

  4. mysql5.7系列还必须要求安装boost_1_59_0(必须是该版本)

    • 本地下载boost_1_59_0, 并且将Tarball包移动到 /usr/local/src 目录下, 并 tar -zxvf mysql-5.7.23.tar.gz 解压;
    • 直接移动解压目录到 /usr/localboost : cp -r boost_1_59_0 /usr/local/boost (不用编译安装)

  5. 开始编译安装

    已经在/usr/local/生成mysql目录

  6. 编译安装完成后, 需要创建运行mysql程序的相关账户及目录相关权限

  7. 接下来需要设置mysql的配置文件my.cnf

    • 注意: mysql5.7.18之后, 貌似已经不在解压包的support-files目录中提供my-default.cnf文件
    • 参考在Unix和类Unix系统上，MySQL程序按照指定的顺序从下表中显示的文件中读取启动选项（首先列出的文件首先读取，后面读取的文件优先）
    • 你会发现在centos7-minimal系统的/etc下就有my.cnf文件
