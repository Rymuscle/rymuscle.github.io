  1. 为什么使用token? 而不使用账号密码?
  • OAuth 的身份验证过程, 其实就是一个获取 access_token 的过程, 为什么要用token而不直接用账号密码来登录?
  • 经常使用 OAuth 身份认证的开发人员, 可能也会感觉到使用 token 都是出现在 让第三方网站访问用户数据提供服务 场景(比如第三方网站接入的微信, 微博登录, 可以提供用户在微信, 微博中的用户数据给第三方站点);
  • 其实 token 与 账号+密码 最大的不同, 就是 token 必然是有时效性的, 并且可被回收, 即使不小心 token 被攻击者获取, 他也没有办法永远能访问你的数据; 而 OAuth 正是提供了让第三方获取 token 这个临时钥匙的通用解决方案。
