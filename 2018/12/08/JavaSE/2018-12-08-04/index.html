可变个数的形参

  1. 格式, 对于方法的形参: 数据类型... 形参名

  2. 可变个数的形参方法与同名方法也构成重载

    • 显然可变个数的形参 方法已经包含了同名方法
    • 可变个数的形参, 在调用时, 参数个数从0开始到无穷多个

  3. 一个方法只能有一个可变个数的形参, 并且可变个数的形参要放在不可变个数形参的后面

  4. 可变个数形参的方法 和 形参为数组 (参数类型要相同) 的方法其实是一模一样的, 所以不能重载

  5. 在调用 可变个数形参 的方法时, 可以传递多个参数, 也可以传递一个数组, 因为 **可变个数形参的方法 和 形参为数组 (参数类型要相同) 的方法是一模一样的

  6. 使用可变个数形参的方法后, 部分方法可以省略

  7. 示例

toString 方法

  1. toString() 在 Object 类中定义, 其返回值是 String 类型, 返回类名和它的引用地址;
  2. 可以根据需要在自定义类中重写 toString()
  3. 基本类型转换为String类型时, 调用了对应包装类的 toString()
  4. 像String类, 包装类, File类, Date类等都实现了Object类中 toString() 的重写
  5. 自定义类的 toString() 也可以像 equals(), 构造器 那样借助IDE生成

基本,包装,String

  1. 基本类型要转换到对应的包装类型: 调用包装类的构造器

  2. 包装类型要转换到对应的基本类型: 调用包装类的方法 xxxValue()

  3. JDK5.0以后, 基本类型和包装类型可以自动拆箱和装箱

  4. 基本类型,包装类型 -> String

    • int i1 = 10; String str1 = i1 + ""; // 即可
    • 调用String类的静态的重载的 valueOf(): String str2 = String.valueOf(i1);

  5. String -> 包装类型 : 调用包装类型的 parseXXX(String str) (对于基本数字类型, 字符串必须能转为数字才行)

  6. String -> 基本类型

    • String 不能直接到基本类型

    • 需要使用 包装类型的 parseXXX(String str) 先转换为包装类型, 然后再用包装类型到基本类型
