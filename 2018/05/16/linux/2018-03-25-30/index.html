30. Linux软件安装及管理

2018-05-16
  • Linux

概述

  1. 在Linux平台下, 软件包的类型可以划分为两类: 源码包、二进制包;

  2. 源码包: 软件的源代码(一般也叫Tarball, 即将软件的源码以tar打包后再压缩的资源包);

  3. 二进制包: 如 Red Hat发行版的 .rpm 包, Debian发行的.deb包;

源码包编译安装

  1. 源码包主要安装步骤:

    • 获取程序源代码文件
    • 编译源代码(如使用gcc编译器进行编译)
    • 生成可执行的二进制文件

  2. 源码包实际安装步骤:

    • 一般会下载软件源码包并解压缩到 /usr/local/src 目录下 (然后可进入源码包查看 INSTALL、README等相关文件)
    • 在源码包中运行 ./configure --prefix=/usr/local/XXX 进行环境检测, 并生成 Makefile文件
      --prefix 是为了指定软件安装目录, 推荐位置为 /usr/local/XXX 下
      如果没有指定–prefix=/path这个参数, 通常默认为/usr/local
    • 编译 make
    • 安装 make install
    • 生成最终软件(包含可执行文件和配置文件等..)

  3. 使用源码包方式编译安装软件, man文件默认保存在/usr/local/man或/usr/local/share/man中, 如果你的安装路径自定义为如/usr/local/XXX, 则man文件则一般是在/usr/local/XXX/share/man中;
