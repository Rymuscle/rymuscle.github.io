Linux文本处理 - awk

2018-05-15
  • Linux

简介

  1. Awk是一种便于使用且表达能力强的程序设计语言, 没错, 它是一门编程语言, 它拥有数组, 支持if-else控制语句, for/while循环语句等功能; 同时, 它是一个强大的文本分析工具;
    在工作中, 你通常可以使用awk对一些文本, 比如日志文件进行一些很方便的分析工作;

  2. 之所以叫AWK是因为其取了三位创始人 Alfred Aho，Peter Weinberger, 和 Brian Kernighan 的Family Name的首字符;

  3. awk 命令的执行过程

    • awk是逐行处理的, 就是说, 当awk处理一个文本时, 会一行一行进行处理, 处理完当前行, 再处理下一行, awk默认以 “换行符” 为标记来识别每一行, 跟我们人类一样, 每次遇到”回车换行”, 就认为是当前行的结束, 新的一行的开始;
    • awk会按照用户指定的分割符去分割当前行为多个字段(每个字段分别被标识为 $1 一直到 $NF, $NF为awk中的变量, 表示当前行被分隔出的最大字段数), 如果没有指定分割符, 默认使用空格作为分隔符, 另外, 默认会将空格符和制表符都视为空格, 而且多个连续的空格都视为一个;

语法结构

  1. 基本语法结构为: awk [options] 'program' file1 , file2 ..., 其中, 'program' 又可以细分为 pattern 和 action; 也就是说, awk的语法为 awk [options] 'pattern {action}' file1 , file2 ...

  2. action 指的就是动作, awk 擅长文本格式化并将格式化后的文本输出, 所以它比较常用的 action 是 print 和 printf;

  3. option 选项, 可以不用任何选项, 常用选项为

    • -F fs or --field-separator fs : 指定输入文件分隔符(), fs可以是一个字符串或是一个正则表达式;
    • -v 变量赋值选项: 该选项将一个值赋予一个变量, 它会在程序开始之前进行赋值, 下面的例子描述了该选项的使用

  4. 多文件处理比较少见, 遇到了再说~~

awk的命令行选项

目前只列出本人目前比较常用的, 稍后会进行补充

  1. awk --version : 输出awk版本信息

  3. -f scriptfile 或者 --file scriptfile : 从脚本文件中读取awk命令;

  1. 默认将空格符和制表符都视为空格, 而且多个连续的空格都视为一个(如果强制指定文件分隔符为多个空格, 效果其实和一个空格一样)

  2. 指定分隔符的时候, 可以使用双引号, 如果分隔符就是双引号, 则需要使用 \"

    • 比如分析linux中的 /etc/passwd 文件时, 你可能需要以 : 为分隔符

      注意, awk 命令中的多个字段需要使用逗号进行分隔, 如: {print $1,$2}, 没有的话, 结果中的两个字段就紧挨着了

  3. -F 还可以使用正则表达式来定制按照某些字符串来做分隔,字符需要放在'[]'中, 比如对nginx的access日志使用[或者]来分隔

awk程序结构

上面已经介绍过了awk的基本语法结构(awk [选项参数] 'awk程序内容' 目标文件s), 并且在介绍awk命令的选项时, 已经提前接触到了一些awk的相关用法, 下面就其程序内容进行学习;

  1. awk程序内容, 需要使用单引号包起来

  2. awk程序的内容通常由模式和动作构成: 'pattern {action}'

    • 模式可以有多个

    • 可以只有模式而没有动作, 也可以只有动作而没有模式, 但是不能两者都没有 (如果没有动作, 则默认与 {print $0} 和 {print} 等价, 为输出匹配行)

    • 注意模式匹配字符串时, 需要使用双引号

  3. 模式可以使用运算符, 也可以使用正则表达式

BEGIN 语句块

  1. BEGIN语句块的语法: BEGIN {awk-commands}: BEGIN语句块在程序开始的使用执行, 它只执行一次, 在这里可以初始化变量。
  2. BEGIN是AWK的关键字, 因此它必须为大写, 注意, 这个语句块是可选的;
